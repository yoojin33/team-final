OWASP TOP 10

웹 보안 위험을 보는 대표 기준

OWASP Top 10은 웹 애플리케이션에서 반복적으로 발견되는 주요 보안 위험을 정리한 목록입니다. Web-Pedia에서는 중간프로젝트 범위에 맞춰 핵심 의미와 대응 방향을 요약했습니다.

취약점 페이지로 돌아가기
A01

Broken Access Control

권한 확인이 부족해 다른 사용자의 정보에 접근하거나 관리자 기능을 실행할 수 있는 위험입니다.

A02

Cryptographic Failures

민감 정보가 암호화되지 않거나 약한 방식으로 보호될 때 발생합니다. 전송과 저장 구간 모두 점검해야 합니다.

A03

Injection

SQL, NoSQL, OS 명령처럼 실행 문맥에 입력값이 섞여 명령 구조가 변조되는 위험입니다.

A04

Insecure Design

설계 단계에서 보안 요구사항과 위협 모델링이 부족해 구조적으로 취약한 기능이 만들어지는 문제입니다.

A05

Security Misconfiguration

기본 계정, 디버그 모드, 불필요한 서비스, 과도한 에러 정보 노출처럼 설정 실수로 생기는 위험입니다.

A06

Vulnerable Components

오래되었거나 알려진 취약점이 있는 라이브러리와 프레임워크를 사용할 때 발생합니다.

A07

Identification and Authentication Failures

인증 로직, 비밀번호 정책, 세션 관리가 약할 때 계정 탈취와 권한 오용으로 이어집니다.

A08

Software and Data Integrity Failures

배포 파일, 업데이트, 직렬화 데이터의 무결성을 검증하지 않아 악성 코드가 섞일 수 있는 위험입니다.

A09

Security Logging and Monitoring Failures

공격 흔적을 기록하거나 탐지하지 못하면 사고가 발생해도 대응 시점을 놓치게 됩니다.

A10

Server-Side Request Forgery

서버가 공격자가 지정한 내부 또는 외부 주소로 요청을 보내도록 유도하는 취약점입니다.

학습 포인트

OWASP Top 10은 암기 목록이 아니라 보안 사고방식의 지도입니다. 각 항목을 실제 코드 작성 시 입력 검증, 권한 확인, 설정 관리, 모니터링으로 연결해 보는 것이 중요합니다.
