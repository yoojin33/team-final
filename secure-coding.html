SECURE CODING

안전한 웹 코딩 체크리스트

보안은 마지막에 추가하는 기능이 아니라, 코드를 작성하는 순간부터 함께 고려해야 하는 개발 습관입니다. 아래 항목을 체크하며 방어 전략을 점검해 보세요.

보안 점검 진행률 0/0 완료

입력 검증

허용할 값의 형식과 길이를 먼저 정의합니다. 금지 목록보다 허용 목록 방식이 더 안전합니다.

출력 인코딩

같은 값이라도 HTML, URL, JavaScript, SQL 문맥마다 필요한 방어 방식이 다릅니다.

최소 권한

사용자, 관리자, API 토큰은 필요한 범위의 권한만 갖도록 분리합니다.

상황별 대응 전략

취약점별로 기억해야 할 방어 포인트를 실무적인 문장으로 정리했습니다.

XSS

출력 위치를 기준으로 방어

사용자 입력을 HTML로 넣어야 한다면 신뢰할 수 있는 sanitize 라이브러리를 사용하고, 일반 텍스트는 textContent로 출력합니다.

SQL

쿼리와 데이터를 분리

Prepared Statement는 입력값을 SQL 명령이 아닌 데이터로 취급하게 만들어 Injection 위험을 줄입니다.

AUTH

인증과 인가를 분리

로그인 여부를 확인하는 것과 특정 리소스에 접근할 권한이 있는지 확인하는 것은 별개의 단계입니다.

CSRF

요청 출처 확인

중요 상태 변경 요청에는 CSRF 토큰을 포함하고, 쿠키 SameSite 옵션을 함께 사용합니다.

FILE

업로드 파일 격리

확장자, 크기, MIME 타입을 확인하고 업로드 파일은 실행 권한이 없는 별도 저장소에 둡니다.

LOG

기록하되 노출하지 않기

보안 이벤트는 서버 로그에 남기고, 사용자에게는 공격자에게 힌트가 되지 않는 일반 메시지를 보여줍니다.

    comment.textContent = userInput;
