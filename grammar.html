WEB STANDARD

웹 표준 제작 문법 가이드

웹 페이지를 만드는 가장 기본 재료인 HTML과 CSS를 기능별로 정리했습니다. 검색과 필터를 이용해 필요한 개념을 빠르게 찾아볼 수 있습니다.

HTML구조

시맨틱 태그

header, nav, main, section, article, footer처럼 의미가 있는 태그로 문서 구조를 표현합니다.

문법 설명

시맨틱 태그는 화면 모양이 아니라 콘텐츠의 역할을 알려줍니다. 검색 엔진, 스크린 리더, 협업자가 문서 구조를 이해하기 쉬워집니다. 

<header>사이트 제목</header>
<nav>주요 메뉴</nav>
<main>
  <section>
    <h2>웹 문법</h2>
    <article>HTML 설명 카드</article>
  </section>
</main>
HTML폼

폼과 입력 요소

label, input, select, textarea를 연결하고 입력 타입과 기본 검증 속성을 사용합니다.

문법 설명

폼은 사용자 입력을 받는 영역입니다. label의 for와 input의 id를 맞추면 클릭 영역과 접근성이 좋아지고, type은 모바일 키보드와 기본 검증에 영향을 줍니다. 

<form>
  <label for="email">이메일</label>
  <input id="email" type="email" required autocomplete="email">
  <button type="submit">제출</button>
</form>
HTML접근성

대체 텍스트와 제목 구조

이미지에는 상황에 맞는 alt를 제공하고, h1부터 h6까지의 제목 계층을 일관되게 구성합니다.

문법 설명

alt는 이미지를 볼 수 없는 사용자에게 정보를 전달합니다. 제목 태그는 글자 크기 조절용이 아니라 문서의 목차 역할로 사용합니다. 

<h1>Web-Pedia</h1>
<h2>웹 보안 취약점</h2>
<img src="image/owasp-logo.png" alt="OWASP 로고">
CSS선택자

선택자와 우선순위

태그, 클래스, 아이디, 속성 선택자의 역할을 구분하고 우선순위를 관리합니다.

문법 설명

CSS는 더 구체적인 선택자가 우선 적용됩니다. 유지보수하기 쉬운 사이트는 대부분 클래스 중심으로 스타일을 작성합니다. 

/* 추천: 역할이 드러나는 클래스 */
.security-card {
  border: 1px solid var(--line);
}

/* 주의: 너무 강한 선택자는 덮어쓰기 어렵다 */
#main .card article h3 { color: red; }
CSS레이아웃

박스 모델

content, padding, border, margin의 차이를 이해하면 요소의 실제 크기와 간격을 예측할 수 있습니다.

문법 설명

box-sizing: border-box를 사용하면 width 안에 padding과 border가 포함되어 레이아웃 계산이 쉬워집니다. 

* {
  box-sizing: border-box;
}

.card {
  width: 320px;
  padding: 24px;
  border: 1px solid #ddd;
  margin: 16px;
}
CSSFlex/Grid

Flexbox와 Grid

Flexbox는 한 방향 정렬, Grid는 행과 열을 모두 다루는 2차원 레이아웃에 적합합니다.

문법 설명

메뉴나 버튼 줄처럼 한 줄 흐름은 flex, 카드 목록처럼 행과 열이 필요한 화면은 grid가 편합니다. 

.nav-menu {
  display: flex;
  gap: 16px;
}

.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
반응형Media Query

반응형 웹

viewport, 상대 단위, media query를 조합해 데스크톱과 모바일에서 모두 읽기 좋은 화면을 만듭니다.

문법 설명

모바일에서는 3열 카드가 너무 좁아지므로 화면 폭에 따라 1열로 바꾸는 식으로 설계합니다. 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

@media (max-width: 760px) {
  .card-grid {
    grid-template-columns: 1fr;
  }
}
JavaScriptDOM

DOM 선택과 이벤트

JavaScript는 HTML 요소를 선택하고 이벤트에 반응해 화면을 동적으로 바꿉니다.

문법 설명

querySelector로 요소를 찾고 addEventListener로 클릭, 입력, 제출 같은 사용자 행동에 반응합니다. 이 사이트의 검색, 필터, 퀴즈도 같은 원리입니다. 

const button = document.querySelector(".theme-toggle");

button.addEventListener("click", () => {
  document.body.classList.toggle("light-theme");
});
JavaScript조건문

변수, 조건문, 반복문

값을 저장하고 조건에 따라 다른 코드를 실행하며, 여러 요소를 반복 처리합니다.

문법 설명

let과 const로 변수를 만들고, if로 조건을 나누며, forEach로 카드 목록 같은 여러 요소를 한 번에 처리할 수 있습니다. 

const cards = document.querySelectorAll(".card");

cards.forEach((card) => {
  if (card.textContent.includes("XSS")) {
    card.classList.add("highlight");
  }
});

참고 기준: MDN HTML, MDN JavaScript Guide

코드로 보는 핵심 문법

grammar-example 
<main class="learning-card">
  <h1>Web-Pedia</h1>
  <p>웹 표준과 보안을 함께 학습합니다.</p>
  <a href="security.html">취약점 보기</a>
</main>
.learning-card {
  display: grid;
  gap: 16px;
  padding: 24px;
  border: 1px solid var(--line);
  border-radius: 8px;
}

문법 학습 팁

  • HTML은 의미 구조를 먼저 잡고, CSS는 그 구조를 꾸미는 역할로 분리합니다.
  • 반복되는 색상, 간격, 글꼴은 CSS 변수로 관리하면 유지보수가 쉬워집니다.
  • 브라우저 개발자 도구로 박스 모델과 적용된 스타일을 확인하는 습관을 들입니다.
